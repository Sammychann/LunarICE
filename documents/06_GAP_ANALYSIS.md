# BAHood (LunarIce-360) — Gap Analysis

This gap analysis was generated by auditing the complete codebase against the Hackathon rules, evaluation criteria, and the project's own Problem Statement.

## 🏆 Current Strengths (Ready for Evaluation)
Our pipeline effectively implements several sophisticated methodologies that strongly align with the hackathon's "Scientific Robustness" and "Innovation" criteria:
- **Multi-Method Fusion Ice Detection:** Combining CPR/DOP thresholds with GMM and Isolation Forest anomaly detection.
- **Dual-Frequency Depth Discrimination:** Differentiating shallow vs. deep ice.
- **Analytical Vectorised Cloude-Pottier Decomposition:** High-performance polarimetric analysis.
- **NSGA-II Rover Traverse Optimization:** Multi-objective planning considering energy, hazards, and shadows.
- **Bayesian MCMC Volumetric Inversion:** Rigorous uncertainty quantification for ice volume.

## 🔴 Critical Gaps & Bugs
1. **UI App Execution Bug:** `ui_app.py` (line 88) currently attempts to run `python -m lunarice360.demo_synthetic`. The project package is `BAHood`, not `lunarice360`. **This will cause the UI demo to fail.**
2. **Physics Bug in Synthetic Demo:** `demo_synthetic.py` (line 376) uses the formula `(S1-S2)/(S1+S2)` for the inline CPR fallback, whereas the correct physics formula is `(S1-S4)/(S1+S4)` (which is correctly used in `polarimetry.py`).

## 🟡 Integration Gaps (ROA Module)
While the new `roa.py` (Region of Attraction) module was successfully implemented, tested (17/17 passing), and partially integrated into `main.py` in the latest `git pull`, some gaps remain:
1. **Visualization:** `visualization.py` lacks a function to plot the ROA (e.g., `plot_roa` or overlaying the ROA mask on the terrain map).
2. **Synthetic Demo:** `demo_synthetic.py` has not been updated to use the new `MemoryAugmentedPlanner` for traverse optimization.
3. **UI Integration:** The Streamlit/Web UI (`ui_app.py`) does not expose the ROA functionality or visualize it for the judges.
4. **`__init__.py`:** Does not list the new `roa` module in its docstring directory.

## 🟠 Code Quality & Maintenance
1. **Missing `__main__.py`:** The package cannot be run via `python -m BAHood` from the command line because it lacks a `__main__.py` file.
2. **Unused Imports:** 
   - `visualization.py`: `mcolors`, `cm`, `Axes3D`
   - `terrain.py`: `generic_filter`
3. **Unused Variables/Parameters:**
   - `traverse.py`: `step_dist` is computed but not used in the accumulation loop.
   - `landing_site.py`: `dem` parameter in `select_landing_sites` is accepted but unused.
4. **Fragile Coupling:** `ice_detection.py` assumes feature indices (0, 1, 5, 7) corresponding to CPR, DOP, H, and alpha. This breaks easily if `polarimetry.build_feature_stack` changes.

## ⚪ Configuration Opportunities
Several hardcoded values could be moved to `config.py` for easier tuning during the hackathon finale:
- **`ice_detection.py`:** GMM sub-sample size (100k), Isolation Forest `n_estimators` (200).
- **`volume_estimation.py`:** Incidence angle (35°), baseline CPR (0.2), calibration factor (0.01).
- **`traverse.py`:** Interpolation count (20).
- **`ui_app.py`:** Server port (8000) and poll interval.
